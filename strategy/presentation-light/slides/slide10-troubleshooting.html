Making It Real: Troubleshooting

The #1 pain point from customer research (opportunity score: 6.1)

Today: Portal-Only Pain

  1. Open the Fabric portal
  2. Navigate workspace, find pipeline, open run history
  3. Read vague error message
  4. Switch tab to Azure Monitor
  5. Search scattered logs across Spark, Pipeline, Eventhouse
  6. Manually correlate timestamps between tools
  7. Google the error, find old community post
  8. Try fix, redeploy, wait, repeat

Hours to resolution. Each tool switch = 23 min lost focus.

Tomorrow: Embedded Experience

Foundation: Unified Logging API + CLI

One API aggregates all log sources. One CLI command gets all relevant logs.

Error enrichment adds root cause context automatically.

Agent Layer: MCP App in VS Code

Pipeline Failure: ETL-Finance-Daily

Root Cause: Memory overflow (87% confidence)

Actions: [Increase memory] [Add partitioning]

User approves action, agent executes the fix and re-triggers the pipeline automatically.
